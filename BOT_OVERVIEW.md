# Краткое описание DocCheck Manager Bot

## Назначение

DocCheck Manager Bot помогает менеджеру сверять заказ поставщику и счет от поставщика.

Бот принимает два Excel-файла, читает товарные строки, сопоставляет позиции по штрихкодам, находит расхождения по количеству, цене и сумме, формирует Excel-отчет и дает краткое резюме через GigaChat.

Бот не принимает решение о согласовании или оплате счета. Финальное решение остается за менеджером.

## На чем построен

Основной стек:

- Python;
- aiogram 3 для Telegram-бота;
- pandas, openpyxl, xlrd для чтения Excel;
- openpyxl для формирования Excel-отчета;
- GigaChat SDK для аналитического резюме и ответов на вопросы;
- python-dotenv для настроек из `.env`;
- локальный JSON-справочник `data/catalog_index.json`.

База данных и миграции в MVP не используются.

## Общая схема работы

1. Администратор заранее готовит справочник:

```powershell
python prepare_catalog.py "Все шк.xlsx"
```

2. Скрипт создает:

```text
data/catalog_index.json
```

3. Пользователь запускает в Telegram команду `/new`.

4. Бот просит загрузить файл заказа.

5. Бот просит загрузить файл счета.

6. Бот читает оба Excel-файла:

- ищет строку заголовков;
- определяет нужные колонки;
- нормализует штрихкоды, числа, цены и суммы;
- учитывает разные шаблоны счетов.

7. Бот сопоставляет строки:

- через `catalog_index.json`;
- по основным и дополнительным штрихкодам;
- напрямую по одинаковому штрихкоду, если штрихкод есть в обоих файлах, но отсутствует в справочнике.

8. Бот сравнивает:

- количество;
- цену;
- сумму;
- скидки;
- НДС и итоговые суммы, если они вынесены отдельными колонками.

9. Бот формирует Excel-отчет.

10. GigaChat формирует структурированное резюме и отвечает на вопросы по текущей сверке.

## Основные файлы и ответственность

### `bot/main.py`

Точка запуска Telegram-бота.

Отвечает за:

- загрузку `.env`;
- создание Telegram Bot API клиента;
- настройку прокси;
- регистрацию команд меню;
- запуск polling.

### `bot/handlers.py`

Telegram-сценарии пользователя.

Отвечает за:

- команды `/start`, `/help`, `/new`, `/reset`, `/catalog_status`;
- прием файла заказа;
- прием файла счета;
- запуск сверки;
- отправку Excel-отчета;
- отправку резюме GigaChat;
- ответы на вопросы пользователя по последней сверке.

### `bot/states.py`

Состояния загрузки файлов.

Используется для понимания, какой файл бот сейчас ожидает:

- заказ;
- счет.

### `prepare_catalog.py`

Подготовка машинного справочника.

Отвечает за:

- чтение исходного Excel-справочника;
- поиск колонок с наименованием и штрихкодами;
- разбор дополнительных штрихкодов через запятую;
- создание индекса `штрихкод -> товар`;
- сохранение `data/catalog_index.json`;
- фиксацию конфликтных штрихкодов.

### `check_documents.py`

Локальный запуск сверки без Telegram.

Пример:

```powershell
python check_documents.py order.xlsx invoice.xlsx
```

Полезен для проверки алгоритма на файлах до запуска Telegram-бота.

### `core/file_reader.py`

Чтение Excel-файлов.

Отвечает за:

- поиск строки заголовков;
- распознавание разных названий колонок;
- поддержку `.xlsx` и `.xls`;
- извлечение EAN из строк вида `Unit EAN: ...`;
- обработку многострочных шапок;
- выбор правильных колонок суммы, цены, скидки и НДС;
- преобразование файла во внутренний список строк.

### `core/normalizer.py`

Нормализация данных.

Отвечает за:

- очистку текста;
- нормализацию штрихкодов;
- приведение чисел к `Decimal`;
- сравнение денежных значений с точностью до копейки.

### `core/catalog.py`

Работа со справочником.

Отвечает за:

- загрузку `data/catalog_index.json`;
- поиск товара по штрихкоду;
- учет конфликтных штрихкодов.

### `core/matcher.py`

Основной алгоритм сверки.

Отвечает за:

- сопоставление заказа и счета;
- определение отсутствующих позиций;
- определение лишних позиций;
- прямое сопоставление по одинаковому штрихкоду, если штрихкода нет в справочнике;
- назначение основных статусов строк.

### `core/price_checker.py`

Проверка цены и суммы.

Отвечает за:

- сравнение цены;
- сравнение суммы;
- проверку ситуаций цена за штуку / цена за упаковку;
- выделение строк для ручной проверки.

### `core/report_builder.py`

Формирование Excel-отчета.

Создает вкладки:

- `Итог`;
- `Расхождения`;
- `Нет в счете`;
- `Лишнее в счете`;
- `Проверить вручную`;
- `Тех_данные`.

### `core/gigachat_client.py`

Интеграция с GigaChat.

Отвечает за:

- структурированное резюме после сверки;
- ответы на вопросы менеджера;
- гипотезы по строкам ручной проверки;
- fallback-резюме, если GigaChat недоступен.

GigaChat не выполняет основное сопоставление и не меняет статусы строк.

### `core/models.py`

Внутренние структуры данных.

Содержит модели:

- строка документа;
- прочитанный документ;
- результат сопоставления строки;
- общий результат сверки.

## Настройки

Настройки лежат в `.env`:

```text
TELEGRAM_BOT_TOKEN=
TELEGRAM_PROXY=
GIGACHAT_CREDENTIALS=
GIGACHAT_SCOPE=
GIGACHAT_MODEL=
GIGACHAT_VERIFY_SSL_CERTS=
```

`.env` не должен попадать в публичный репозиторий.

## Временные файлы

Загруженные пользователем файлы временно сохраняются в:

```text
temp/uploads/
```

Excel-отчеты временно создаются в:

```text
temp/reports/
```

После успешной отправки бот удаляет временные файлы.

## Что важно помнить

- Для лучшего результата в Excel нужно оставлять только товарные строки.
- Справочник должен быть актуальным.
- Если штрихкод есть в обоих документах, но отсутствует в справочнике, бот может сопоставить строки напрямую.
- Если данных недостаточно или расчет противоречивый, строка попадает в ручную проверку.
- GigaChat помогает анализировать результат, но не принимает решения за менеджера.
