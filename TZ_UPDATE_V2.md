# DocCheck Manager Bot  
## ТЗ на доработку отчёта сверки документов v2 для Codex

## 1. Контекст

В проекте **DocCheck Manager Bot** уже реализована сверка заказа поставщику и счёта от поставщика.

По итогам первого тестирования менеджером нужно доработать **Excel-отчёт**:

1. добавить расчётные колонки по количеству;
2. добавить расчётные колонки по сумме;
3. скрывать незначимые денежные отклонения;
4. объединить вкладки с отсутствующими и лишними позициями;
5. добавить статус позиции в объединённую вкладку.

Это ТЗ относится только к доработке отчёта.  
Не нужно переписывать весь бот заново.

---

## 2. Главное ограничение

В рамках этой задачи **не менять**:

1. алгоритм сопоставления товаров;
2. работу со справочником `catalog_index.json`;
3. работу со штрихкодами;
4. работу GigaChat;
5. Telegram-интерфейс;
6. структуру загрузки файлов;
7. логику OCR;
8. интеграции;
9. общую архитектуру проекта.

Нужно доработать только формирование результата и Excel-отчёта.

## 3. Задача 1. Добавить колонку «Разница по количеству»

### Что сделать

Во вкладку `Расхождения` добавить колонку:

```text
Разница по количеству
```

### Формула

```text
Разница по количеству = Количество в счёте − Количество в заказе
```

### Пример

| Кол-во заказ | Кол-во счёт | Разница по количеству |
|---:|---:|---:|
| 10 | 8 | -2 |
| 10 | 12 | 2 |
| 10 | 10 | 0 |

### Зачем

Чтобы менеджер сразу видел, где поставщик выписал больше или меньше товара.

---

## 4. Задача 2. Добавить колонку «Разница по сумме, ₽»

### Что сделать

Во вкладку `Расхождения` добавить колонку:

```text
Разница по сумме, ₽
```

### Формула

```text
Разница по сумме, ₽ = Сумма счёта − Сумма заказа
```

### Пример

| Сумма заказ | Сумма счёт | Разница по сумме, ₽ |
|---:|---:|---:|
| 1000 | 1100 | 100 |
| 1000 | 950 | -50 |
| 1000 | 1000 | 0 |

### Зачем

Менеджеру важно сразу видеть денежную разницу по каждой позиции.

---

## 5. Задача 3. Добавить колонку «Разница по сумме, %»

### Что сделать

Во вкладку `Расхождения` добавить колонку:

```text
Разница по сумме, %
```

### Формула

База процента — сумма заказа.

```text
Разница по сумме, % = ((Сумма счёта − Сумма заказа) / Сумма заказа) × 100
```

### Пример

| Сумма заказ | Сумма счёт | Разница по сумме, ₽ | Разница по сумме, % |
|---:|---:|---:|---:|
| 1000 | 1100 | 100 | 10% |
| 1000 | 950 | -50 | -5% |

### Правило для нулевой или отсутствующей суммы заказа

Если сумма заказа равна 0 или отсутствует, процент не считать.

В ячейке можно указать:

```text
н/д
```

или оставить ячейку пустой.

---

## 6. Задача 4. Ввести порог отображения денежных расхождений 0,5%

### Что сделать

Добавить правило:

```text
Если абсолютная разница по сумме меньше 0,5%, позицию не выводить как денежное расхождение.
```

### Формула проверки

```text
abs(Разница по сумме, %) < 0,5%
```

Такие строки не должны попадать во вкладку `Расхождения`, если единственная причина расхождения — небольшая денежная разница.

### Важно

Порог **0,5% применяется только к расхождениям по сумме / цене**.

Порог **не применяется** к расхождениям по количеству.

То есть если количество отличается, позиция должна попасть в отчёт независимо от процента денежной разницы.

### Примеры

| Кол-во отличается | Разница по сумме, % | Попадает в расхождения? | Причина |
|---|---:|---|---|
| Нет | 0,3% | Нет | Денежное отклонение меньше 0,5% |
| Нет | 0,7% | Да | Денежное отклонение больше 0,5% |
| Да | 0,1% | Да | Количество отличается, порог не применяется |

## 7. Задача 5. Объединить вкладки «Нет в счёте» и «Лишнее в счёте»

### Что сделать

Вместо двух отдельных вкладок:

```text
Нет в счёте
Лишнее в счёте
```

создать одну общую вкладку:

```text
Нет/Лишнее в счёте
```

### Зачем

Менеджеру удобнее видеть отсутствующие и лишние позиции рядом, чтобы искать возможную замену, ротацию или ошибку поставщика.

### Важно

После доработки старые отдельные вкладки больше не создавать.

---

## 8. Задача 6. Добавить колонку «Статус» в объединённую вкладку

### Что сделать

Во вкладке:

```text
Нет/Лишнее в счёте
```

добавить колонку:

```text
Статус
```

### Допустимые значения

Только два значения:

```text
Нет в счёте
Лишнее в счёте
```

### Логика

Если позиция есть в заказе, но не найдена в счёте:

```text
Статус = Нет в счёте
```

Если позиция есть в счёте, но не найдена в заказе:

```text
Статус = Лишнее в счёте
```

## 9. Обновлённая структура Excel-отчёта

После доработки отчёт должен содержать вкладки:

```text
Итог
Расхождения
Нет/Лишнее в счёте
Проверить вручную
Тех_данные
```

Вкладки:

```text
Нет в счёте
Лишнее в счёте
```

отдельно больше не создавать.

## 10. Обновлённая вкладка «Расхождения»

Во вкладке `Расхождения` должны быть колонки:

```text
Статус
Наименование
Артикул заказа
Артикул счёта
Штрихкод заказа
Штрихкод счёта
Кол-во заказ
Кол-во счёт
Разница по количеству
Цена заказ
Цена счёт
Сумма заказ
Сумма счёт
Разница по сумме, ₽
Разница по сумме, %
Комментарий
```

### Правила попадания строки во вкладку «Расхождения»

Строка попадает во вкладку `Расхождения`, если выполняется хотя бы одно условие:

1. отличается количество;
2. отличается сумма на 0,5% или больше;
3. отличается цена, и это приводит к значимой разнице по сумме;
4. строка имеет статус, требующий проверки по цене или сумме.

Строка не должна попадать во вкладку `Расхождения`, если единственное отличие — денежное отклонение меньше 0,5%.

## 13. Критерии готовности

Доработка считается выполненной, если:

1. В отчёте появилась колонка `Разница по количеству`.
2. Разница по количеству считается как:

```text
количество в счёте − количество в заказе
```

3. В отчёте появилась колонка `Разница по сумме, ₽`.
4. Разница по сумме считается как:

```text
сумма счёта − сумма заказа
```

5. В отчёте появилась колонка `Разница по сумме, %`.
6. Процент считается от суммы заказа.
7. Если сумма заказа равна 0 или отсутствует, процент не считается.
8. Если денежная разница меньше 0,5%, строка не выводится как расхождение.
9. Порог 0,5% не применяется к расхождению количества.
10. Вкладки `Нет в счёте` и `Лишнее в счёте` объединены.
11. Новая вкладка называется:

```text
Нет/Лишнее в счёте
```

12. В объединённой вкладке есть колонка `Статус`.
13. В колонке `Статус` используются только значения:

```text
Нет в счёте
Лишнее в счёте
```

14. Старые отдельные вкладки `Нет в счёте` и `Лишнее в счёте` больше не создаются.
15. Вкладка `Итог` показывает количество скрытых мелких денежных расхождений.
16. Существующая логика сверки по штрихкодам, количеству, цене, справочнику и GigaChat не ломается.
